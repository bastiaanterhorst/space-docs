Space includes an integrated MCP (Model Context Protocol) server that lets agentic AI tools read and optionally edit your Space data.
This enables exciting workflows that combine Space with the other systems that you use. 

You can use it to:

- Read inbox/day/week/project/area views.
- Search tasks and inspect long-term plans.
- Create, update, sort, and delete tasks/projects/areas/groups (when write mode is enabled).
- Apply bulk task changes.
- Manage yearly and quarterly plans from AI.
- Use built-in prompts/completions and read-only MCP resources.

---

## Enabling MCP in Space

<Info>MCP support is included in the Mac version of Space.</Info>

In Space:

1. Open `Settings`.
2. Go to `AI`.
3. Turn on `Enable MCP`.

In this section you can configure:

- `Port` (default: `2402`, valid range: `1024-65535`).
- `Access Mode` (`Read Only` or `Read & Write`).

### Access Mode: Read Only

Use this when you want safe visibility from AI tools.

- AI can read your views, projects, areas, groups, and long-term plans.
- Write tools are not exposed.
- Any write attempt is rejected.

### Access Mode: Read & Write

<Warning>When you enable Read & Write your AI tools can manipulate your data in Space, including deleting data. Start with read-only mode first and move to write mode only when you trust your tool setup.</Warning>

Use this when you want the AI to actively manage your Space data.

- Includes all read tools.
- Adds write tools for task/project/area/group/plan management.
- Destructive actions require confirmation:
  - Interactive confirmation when the MCP client supports elicitation.
  - Or explicit `confirm_destructive: true` when interactive confirmation is unavailable.

## Connecting AI tools to Space

In `Settings > AI`, copy the `Endpoint` shown in the `Authenticate` section (for example `http://127.0.0.1:2402/`) and use that as your MCP server URL in your AI tool.

Space MCP listens on localhost (`127.0.0.1`) and uses HTTP transport with OAuth. 
The oAuth flow will ask for a time based pairing code which is shown in the settings screen.

### Connecting to Claude Code

In a terminal, run `claude mcp add --transport http space http://127.0.0.1:2402/`.
Then start Claude Code by running `claude`. When started, run `/mcp`. You should see Space listed as 'unauthenticated'. Press enter to select Space, and then again enter to confirm authentication.
A browser window will appear asking for your pairing code. Get it from your Space Settings screen on the AI tab.

<Frame caption="Claude inspecting Space MCP">
  <img src="/images/mcp/claude.png" />
</Frame>

### Connecting to OpenAI Codex CLI

In a terminal, run `codex mcp add Space --url http://127.0.0.1:2402/`. A browser window should open asking you for the pairing code. 
Get it from your Space Settings screen on the AI tab.

The Codex Desktop app will automatically pick this registration up as well!

### Connecting to OpenAI Codex Desktop App

Open `Settings` -> `MCP Servers` and click `+ Add server`. Give the server a name (like `Space`). Select `Streamable HTTP` as the transport and enter `http://127.0.0.1:2402/` as URL.

<Frame caption="Codex Desktop">
  <img src="/images/mcp/codex-ui-1.png" />
</Frame>

Click `Save`. Now you see an `Authenticate` button appear next to the entry. Click it and enter the pairing code found in Space Settings on the AI tab.

### Connecting to Craft Agents

Navigate to `Sources` -> `MCPs` and click `+`. In the popup, enter something like "Connect to "Space Life Planner", an MCP server at http://127.0.0.1:2402/"
After some noodling, Craft Agents will ask you to allow opening the oAuth screen. Do that and enter the pairing code you find in Space Settings on the AI tab.


### Connected Clients

Space shows all connected clients. You can always disconnect a client from within Space to immediately revoke its access.

## What You Can Do

### Read operations

- `get_inbox`, `get_day`, `get_day_events`, `get_week`, `get_project`, `get_area`
- `list_projects`, `list_areas`, `list_groups`
- `list_long_term_plans`, `get_long_term_plan`
- `search_todos`

### Write operations

Task operations:

- `add_task`, `edit_task`, `delete_task`, `sort_tasks`
- `bulk_edit_tasks`, `bulk_delete_tasks`
- `schedule_task_event`

Structure operations:

- `add_project`, `edit_project`, `delete_project`
- `add_area`, `edit_area`, `delete_area`
- `add_group`, `edit_group`, `sort_groups`, `delete_group`

Planning metadata and cleanup:

- `edit_day`, `edit_week`
- `delete_completed_todos` (project/area scope)

Long-term planning (yearly and quarterly plans):

- `add_long_term_plan`
- `edit_long_term_plan`
- `list_long_term_plans` (defaults to active plans when no state filter is passed)

## 7. Prompts and Resources

You can use Space with tools only. Prompts and resources are optional, but useful in clients that support them.

### Prompts

Use prompts when you want Space to package the right data plus a strong instruction template for specific workflows.

Prompt families:

- Briefings (quick overview): `day_brief`, `week_brief`
- Planning (actionable plans): `day_plan`, `week_plan`
- Operational review: `inbox_triage`, `project_review`
- Long-term planning: `ltp_summary`, `ltp_status_update`, `ltp_next_actions`, `ltp_risks`, `ltp_execution_alignment`, `ltp_plan_assist`

How to think about them:

- Use a **briefing** prompt when you want a quick summary of your day or week.
- Use a **planning** prompt when you want the AI to propose an execution plan.
- Use `ltp_plan_assist` when you want help designing a new quarter/year plan.

### Resources

Use resources when your client can preload read-only snapshots before generating a response. This usually improves answer quality with less manual tool-calling.

Resources are read-only snapshots such as:

- `space://context/today`
- `space://context/this_week`
- `space://context/day/{date}`
- `space://context/week/{year}/{week}`
- `space://plans/active`
- `space://projects/{project_id}/summary`

If your client does not support resources, you can ignore them and call tools directly (`get_day`, `get_week`, `get_day_events`, etc.).

## Important Behavior Notes

- `day_iso` values are local calendar day semantics (`YYYY-MM-DD`) in the current system timezone.
- Week inputs use ISO week semantics (`week` + `year`).
- `get_day_events` uses the same visible-calendar pipeline as the in-app calendar.
- Tools tht offer `include_late` / `include_late_day` filtering fall back to Space preferences when omitted.
- `sort_tasks` follows your rollover preference for day/week late-task visibility. Sorting a day or week also normalizes visible late tasks to that day/week.
- Group `intensity` is a score from `0` to `10`; use `null` in `edit_group` to disable group intensity scoring. Group intensity is primarily used for yearly and quarterly plans.
- IDs such as `project_id`, `area_id`, `group_id`, and `plan_id` are opaque tokens returned by Space.

## Recommended Setup

1. Start in `Read Only`.
2. Connect your MCP client to the HTTP endpoint shown in Space.
3. Complete OAuth using the pairing code shown in `Settings > AI`.
4. Validate basic reads first (`get_day`, `get_week`, `get_inbox`).
5. Move to `Read & Write` only after your AI behavior is predictable.
6. Keep delete confirmation enabled in your agent/tool flow (`confirm_destructive` fallback where needed).

This gives you strong safety while still unlocking practical MCP automation.
